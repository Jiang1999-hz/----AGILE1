## 当前 Demo 已包含的核心模块

## Student S1 当前已开始接入真实数据

- 学生端登录后会从本地后端读取真实数据
- 当前后端 API 已切到 `Prisma + Supabase PostgreSQL`
- `data/student-s1.json` 现在主要用于 seed 数据导入
- 已支持的后端接口能力：
  - 读取学生课程与课次
  - 读取学习记录
  - 提交在线做题结果
  - 持久化保存结果到数据库

这意味着学生端已经不只是纯前端展示，而是开始具备真实数据闭环。

### 1. 假登录系统

- 学生
- 老师
- Admin

三种角色登录后都会先进入统一的 `塾 News` 页面。

### 2. 学生端

- 课程模块
  - 同时展示多门课程，例如：
    - 早稻田大学入试备考课程
    - 日语精读课程
    - 英语读解强化课程
  - 每门课展示总课次、已完成课次、进度条、时间安排、下节课、老师
- 课程日历
  - 点击课程后进入对应课程日历
  - 再点击单节课查看详细内容
- 单节课详情
  - PPT
  - 课堂笔记
  - 重难点
  - 易错点
  - 完成度
  - 理解度
  - 作业状态
  - 错题数量
  - 本节课表现图表
- 在线做题
  - 自动评分
  - 自动更新学习进度
  - 自动更新老师反馈与家长摘要
- 我的学习情况
- 和老师沟通
- 和 AI 沟通

### 3. 老师端

- 学生数据库
  - 左侧学生 data table
  - 右侧学生详情边栏
  - 风险状态、到课、最近测验、最近作业等信息
- 学生详情与反馈
  - 每次作业情况
  - 写老师反馈
  - 生成家长版反馈
- 教学助手
- 课程模块 / 课程日历
  - 先看课程包
  - 再看课程日历
  - 再点单节课看详情

### 4. Admin 端

- 经营概览
- 班级与老师
- 缴费与预警
- 系统配置
- 风险学生与服务质量观察

## 当前适合拿去讲的价值

- 把学生做题、课程、课后反馈沉淀成可复用数据
- 帮老师减少批改、整理、写反馈和准备面谈的时间
- 帮学生看清自己同时在学哪些课程、每门课进度如何
- 帮家长更容易理解每节课学了什么、孩子掌握得怎么样
- 帮塾长更早发现风险学生、课程进度和经营问题

## 推荐演示顺序

1. 登录任意角色，先看统一的 `塾 News`
2. 用学生身份演示：
   - 课程模块
   - 某门课的课程日历
   - 单节课详情
   - 在线做题
   - 和老师 / AI 沟通
3. 切换老师身份演示：
   - 学生数据库
   - 学生详情与反馈
   - 老师课程模块与课程日历
4. 切换 Admin 身份演示经营概览
