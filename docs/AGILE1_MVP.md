# Agile 1 MVP 定义

## 当前目标

做一个可以直接拿给小私塾负责人、老师、朋友演示的多角色 Demo，证明这套产品具备以下价值：

1. 教学过程可视化
2. 学生学习进度可追踪
3. 每节课内容可沉淀
4. 老师工作可提效
5. 私塾运营可被更早预警

## Agile 1 当前已覆盖的重点功能

### P0

1. 多角色假登录
- 学生
- 老师
- Admin

2. 共通 News 页面
- 所有角色进入后先看到塾内公告与运营动态

3. 学生端学习闭环
- 在线做题
- 自动评分
- 自动更新学习进度
- 展示个人画像与反馈
- 和老师沟通
- 和 AI 沟通

4. 课程模块
- 一个学生可同时拥有多门长期课程
- 每门课程展示：
  - 总课次
  - 已完成课次
  - 进度条
  - 时间安排
  - 下节课时间
  - 老师

5. 课程日历与课次详情
- 点击课程进入对应课程日历
- 点击单节课进入课程详情
- 单节课详情展示：
  - PPT
  - 笔记
  - 重难点
  - 易错点
  - 完成度
  - 理解度
  - 作业状态
  - 错题数
  - 表现图表

6. 老师端后台
- 学生数据库
- 左侧 data table + 右侧详情边栏
- 学生作业记录
- 教师反馈
- 家长反馈生成
- 老师课程模块与课程日历

7. Admin 端后台
- 经营概览
- 班级与老师
- 缴费与预警
- 系统配置
- 风险学生观察

## 这版 Demo 主要打的痛点

1. 学生会同时学多门课，但缺少统一进度视图。
2. 老师很难把每节课的内容沉淀成可回看的资产。
3. 学生做题数据和课次内容没有汇总到一起。
4. 家长难以理解孩子每节课到底学了什么、掌握得如何。
5. 私塾负责人很难同时看到教学过程和经营预警。

## 当前演示逻辑

1. 先看课程模块
2. 再看课程日历
3. 再看单节课详情
4. 再看学生做题和反馈闭环
5. 最后切到老师端和 Admin 端

## Agile 1 验收标准

- 可以登录不同角色
- 可以查看多门课程
- 可以点击课程进入日历
- 可以点击单节课进入详情
- 可以做题并更新进度
- 可以让老师查看学生和课次信息
- 可以让 Admin 看到基础经营信息
